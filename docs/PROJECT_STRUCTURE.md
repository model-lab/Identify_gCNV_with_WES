# Project structure (standardized)

This repository uses a clear layout for a Snakemake-based CNV pipeline.

- `workflow/` — Snakemake workflow, `config.yaml` and smoke tests.
- `data/` — reference FASTA, interval lists, prepared inputs; data used by rules.
- `model/` — trained models used by callers (baseline-model, ploidy-model).
- `plans/` — per-sample YAML plan files generated by `scripts/plan_agent.py`.
- `scripts/` — helper scripts (plan agent, download helpers, etc.).
- `outputs/` — suggested output directory for final results (created by workflow runs).
- `.github/` — CI workflows (now points at `workflow/smoke/Snakefile` for smoke tests).

Notes:

- Run the workflow from the `workflow/` directory. `workflow/config.yaml` contains repo-relative defaults.
- The `scripts/plan_agent.py` now reads `workflow/config.yaml` for overrides.
- Keep large genome files out of Git (use Git LFS or public Zenodo links); CI runs a minimal smoke test only.
